# SAIREN VM-Q1 接続検証レポート

**作成日:** 2026-03-15
**対象環境:** Windows 11 Pro / Raspberry Pi Zero 2W（予定）

---

## 1. 調査目的

SAIREN VM-Q1 マイクが正常に動作・録音できるかを検証し、
Raspberry Pi Zero 2W での利用可否を調査する。

---

## 2. デバイス認識状況

### Windows 側での認識

| 項目 | 内容 |
|---|---|
| デバイス名 | KM_B2 Digital Audio |
| VID / PID | VID_8087 / PID_1024 |
| デバイスクラス | USB Audio (MEDIA) |
| ステータス | OK |
| AudioEndpoint（入力） | マイク (KM_B2 Digital Audio) |
| AudioEndpoint（出力） | スピーカー (KM_B2 Digital Audio) |

**補足:**
`スピーカー` エンドポイントが存在するのは正常。USB オーディオデバイスは入力・出力の両エンドポイントを持つ。
VM-Q1 自体はアナログマイクであり、間に挟んだ **音声端子-USB 変換アダプター** が KM_B2 として認識されていた。

---

## 3. 当初の接続構成と問題

### 問題のあった接続（録音不可）

```
VM-Q1 (3.5mm TRS)
  → 延長ケーブル
    → 音声端子-USB 変換アダプター (KM_B2)
      → TypeC to USBa アダプター
        → USB ハブ
          → PC
```

**症状:**
ffmpeg / sounddevice / waveIn API のいずれも録音データが 0 バイト。
中継機器が多すぎることによるシグナルロスが原因と判断。

### 改善後の接続（動作確認）

```
VM-Q1 (3.5mm TRS)
  → 音声端子-USB 変換アダプター (KM_B2)
    → PC（USB 直刺し）
```

**結果:** 録音成功（882,000 bytes / 10 秒）

---

## 4. 録音テスト結果

### 使用 API

Windows waveIn API（ctypes 経由）

### 結果

| 項目 | 値 |
|---|---|
| サンプルレート | 44,100 Hz |
| チャンネル | モノラル（1ch） |
| ビット深度 | 16 bit |
| 録音時間 | 10 秒 |
| ファイルサイズ | 882,000 bytes |
| 最大振幅 | 1,716 / 32,767 (5.2%) |
| RMS | 39.8 |
| 無音比率 | 99.9% |

**判定:** マイクは動作中。ただしゲインが極めて低い。

**原因の仮説:**
VM-Q1 はコンデンサーマイクであり **プラグインパワー（約 2〜3V）が必要**。
USB オーディオアダプター（KM_B2）がプラグインパワーを供給できていない可能性がある。

## 5. VM-Q1 製品仕様

| 項目 | 内容 |
|---|---|
| 型番 | SAIREN VM-Q1 |
| マイク方式 | エレクトレットコンデンサー |
| 指向性 | カーディオイド |
| 周波数特性 | 100 Hz〜20 kHz |
| 感度 | -38.0 dB（1kHz, 1Pa） |
| S/N 比 | 70 dB |
| 電源 | **プラグインパワー必須**（電池不要） |
| 出力端子 | 3.5mm **TRS**（カメラ用） |
| 付属ケーブル | TRS→TRRS 変換ケーブル（スマホ用） |

### 接続先別プラグインパワー供給

| 接続先 | プラグインパワー供給 | 動作可否 |
|---|---|---|
| 一眼カメラ（Sony/Canon 等） | あり | 動作する |
| スマートフォン（TRRS） | あり | 動作する |
| PC ピンクマイク端子（Realtek） | **機種依存** | 不安定な場合あり |
| USB オーディオアダプター | **アダプター依存** | 要確認 |
| Raspberry Pi Zero 2W（3.5mm） | **出力専用のため不可** | 使用不可 |

---

## 6. Raspberry Pi Zero 2W での利用について

### 結論：3.5mm 直結は不可

Pi Zero 2W の 3.5mm ジャックは **出力専用**。
SoC レベルの制約であり、ファームウェア・ドライバーでの解決は不可能。

### 推奨接続方法

#### 方法1: USB オーディオアダプター経由（推奨・最も簡単）

```
VM-Q1 (3.5mm TRS)
  → 音声端子-USB 変換アダプター (KM_B2)
    → micro USB OTG アダプター
      → Raspberry Pi Zero 2W
```

- Pi Zero 2W の micro USB ポートは USB OTG 対応
- Linux カーネルは USB Audio Class デバイスを標準サポート
- 追加ドライバー不要

#### 方法2: I2S MEMS マイクモジュール（GPIO 接続）

| 製品例 | 接続 | 備考 |
|---|---|---|
| INMP441 | I2S (GPIO) | 高音質・ハンダ付け必要 |
| SPH0645LM4H | I2S (GPIO) | 同上 |
| ReSpeaker 2-Mics HAT | GPIO HAT | 簡単だがコスト高 |

---

## 7. 今後の推奨アクション

1. **KM_B2 アダプターを捨てずに保管する**
   → Pi Zero 2W での USB 接続に再利用できる

2. **PC での録音ゲイン改善**
   → Windows サウンド設定でマイクブーストを +20 dB に設定
   → または KM_B2 アダプターがプラグインパワーを供給しているか確認

3. **Pi Zero 2W での動作確認手順**
   ```bash
   # デバイス確認
   arecord -l

   # 録音テスト（10秒）
   arecord -D plughw:1,0 -f S16_LE -r 44100 -d 10 test.wav

   # 再生確認
   aplay test.wav
   ```

## 8. まとめ

| 項目 | 結果 |
|---|---|
| デバイス認識 | OK（KM_B2 として認識） |
| USB 直刺しでの録音 | 成功（ただしゲイン低） |
| 延長ケーブル等経由での録音 | 失敗（シグナルロス） |
| PC ピンクマイク端子への直刺し | プラグインパワー不足のため実用困難 |
| Pi Zero 2W（3.5mm）への直刺し | **不可**（出力専用） |
| Pi Zero 2W（USB OTG 経由） | **可能**（KM_B2 アダプター再利用） |
